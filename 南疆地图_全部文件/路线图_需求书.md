# 自驾路线图制作需求书

## 一、概述

基于 Python + Cartopy + Matplotlib 生成自驾路线图，输出为高分辨率 PNG 图片。地图包含真实地理底图、省份边界、城市节点、路线轨迹、天次标注、景点标注、距离/时间标注等完整信息。

---

## 二、底图要求

### 2.1 地理要素
| 要素 | 规格 |
|:----|:----|
| 陆地 | 浅米色 `#F8F4EC` |
| 水域 | 浅蓝色 `#E8EDF4` |
| 湖泊 | 填色 `#DAE5F0`，边框 `#AABBCC` |
| 海岸线 | `#999999`，线宽 1.2 |
| 经纬网格 | 虚线，不干扰主信息，仅显示左/下刻度 |
| 投影 | PlateCarree（等距圆柱投影） |
| 图片尺寸 | 22×26 英寸，120 DPI |
| 背景色 | 纯白 |

### 2.2 省份边界
- 只显示途经省份的边界线
- 每条省界线用对应省份颜色，线宽 2.0，透明度 0.25
- 省份名称以半透明水印印在地图上（透明度 0.30），既可辨识又不干扰主信息
- 数据来源：Natural Earth admin_1_states_provinces（50m 精度）

## 三、路线线条

| 属性 | 规格 |
|:----|:----|
| 线宽 | 粗线条 + 多层发光羽化 |
| 发光效果 | 多层羽化光环 |
| 颜色 | **每天独立一种颜色，相邻天次颜色不可接近**。调色板应支持任意天数，不限于 8 天。 |

| 属性 | 规格 |
|:----|:----|
| 形状 | 正圆形 |
| 填充色 | 白色 |
| 边框色 | 按省份区分 |
| 城市名 | **放在圆圈内部正中央** |
| 城市名字体 | **大号加粗黑体** |
| 城市名颜色 | 按省份色 |
| 同纬度城市 | 手动错开避免重叠 |

## 五、字体规范

### 5.1 通用规则
| 属性 | 规格 |
|:----|:----|
| 首选字体 | 华文黑体 STHeiti Medium.ttc（macOS 系统字体） |
| 备选字体 | 兰亭黑 Lantinghei.ttc |
| 全部文字 | **加粗（weight='bold'）** |
| 白色描边 | 所有文字加白色描边，确保在线路/底色上清晰 |

### 5.2 各类文字规格
| 类型 | 大小 | 颜色 | 对齐 |
|:----|:----:|:----:|:----:|
| 主标题 | 大号 | 深色 | 居中 |
| 副标题 | 中号 | 灰色 | 居中 |
| 城市名（圈内） | 大号 | 按省份色 | 居中 |
| D1-DN 天次 | 大号 | **统一紫色** | 底部居中 |
| 距离 | 中大号 | 蓝色 | 灵活 |
| 车程时间 | 中大号 | 橙色 | 灵活 |
| 主要景点 | 大号 | 棕色 | 引线末端 |
| 次要景点 | 中大号 | 绿色 | 引线末端 |

## 六、天次标注

### 6.1 非休息日
- **位置**：一天跨多段的，尽量标到**各段总和的中间位置**；所有标注尽量靠近线条但优先避开其他元素
- **优先级**：靠近线路 ← 但优先避开其他元素，保证**不重叠**
- **颜色**：全部天次统一用紫色

### 6.2 休息日
- **位置**：必须设在休整城市的**正下方、靠近城市**
- **格式**：`D4（洛阳全天）` 形式，D* 醒目、括号内容清晰即可
- **颜色**：与 D1-DN 统一紫色

---

## 七、距离/时间标注

- **位置**：靠近对应线路线条，但不能叠加到线条上；同时**尽量避开其他元素**（天次、景点、城市圆圈等）
- **布局**：距离和时间不必须放在一起，必要时可以分开显示到不同位置，但两者要**尽量靠近**以便阅读
- **时间格式**：用中文"小时"（如"2.5小时"），不可用"h"
- **距离格式**：用"km"（如"200km"）
- **每段都标**

---

## 八、景点标注（引线）

| 属性 | 主景点 | 次要景点 |
|:----|:----:|:----:|
| 引线类型 | **实线** | **虚线** |
| 引线起点 | 实心圆点 | 空心圆点 |
| 文字颜色 | 棕色 | **绿色** |
| 文字位置 | 引线末端 |
| 左右交替 | 相邻城市交替左右布局，避免文字打架 |
| 间距 | 主/次景点纵向间隔足够，不重叠 |
| 引线长度 | 必须保证最长景点名称不回流撞到城市圆圈 |

---

## 九、**核心约束（所有元素禁止重叠）**

以下所有元素之间均不得重叠：

- 路线线条 ↔ 路线线条
- 路线线条 ↔ 任何文字
- 路线线条 ↔ 城市圆圈
- 文字 ↔ 文字（天次、距离、时间、景点、城市名等）
- 文字 ↔ 城市圆圈
- 城市圆圈 ↔ 城市圆圈

> 当"靠近线路"和"不压线"冲突时，**不压线优先**。

## 十二、数据结构（可复用模板）

城市数据格式：
```
(名称, 纬度, 经度, 主要景点, 次要景点, 省份代码)
```

路段数据格式：
```
(起点, 终点, 天次, 距离文字, 时间文字)
```

休息日数据格式：
```
{天次: 城市名}
```

省份颜色映射：灵活定义

天次颜色映射：支持任意天数，相邻天次颜色有足够区分
