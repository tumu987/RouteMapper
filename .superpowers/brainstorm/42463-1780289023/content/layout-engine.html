布局引擎：6层逐层放置 + 密度感知

每层按优先级顺序放置，后层主动避开前层所有已占空间

核心流程

Layer Pipeline
L1 底图 → L2 路线 → L3 城市圈 → L4 天次 → L5 距离/时间 → L6 景点+行程表

各层详细策略

L1 底图 + L2 路线线条

📍 地理坐标固定，路线注册到碰撞系统（线宽 + 羽化光环半径）

Natural Earth 底图 → 省界 → 路线多层发光

L3 城市节点

📍 地理坐标固定，注册椭圆碰撞区

圆圈 + 圈内居中城市名 + 同纬度自动微调偏移（如平顺/长治）

L4 天次标注（D1-D14）⏤ 密度感知

行车段：放路线中点法线方向 → 扫描两侧已有元素密度 → 选稀疏侧 → 从最近距离开始尝试

休息日：城市正下方放置 "D3（喀什全天）"，逐步向下搜索

关键：天次优先级高于距离/时间，后者会主动避让天次

L5 距离/时间标注 ⏤ 避让天次

先检查天次放哪侧 → 主动去对面 → 优先上下叠放（距离在上，时间紧挨下方）

如果对面也放不下，允许分开距离和时间到不同位置，但尽量靠近

L6 景点引线 + 行程表 ⏤ 最后填充

景点：城圈内的合并引线拉出；城圈外的打点+紧邻标签。左右交替，8方向寻空

行程表：网格搜索地图上最空的矩形区域，优先左上角

碰撞检测：矩形精确碰撞

检查对象 碰撞类型 Margin
文字 ↔ 文字 矩形 vs 矩形 0.04°
文字 ↔ 城市圈 矩形 vs 椭圆 0.04°
文字 ↔ 路线 点到线段距离 0.06°
同层避让（天次↔天次） 矩形 vs 矩形 0.02°（宽松）
对侧元素（天次 vs dist） 矩形 vs 矩形 0.01°（极宽松）

💡 关键技巧：天次和距离/时间如果在路线对侧，margin 降到 0.01°，几乎不互相排斥，只防止真正重叠
