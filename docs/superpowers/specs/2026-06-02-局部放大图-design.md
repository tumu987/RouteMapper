# 局部放大图（Zoom Inset）设计文档

## 背景

北疆路线中，喀纳斯与禾木相距仅约 18km（约 0.15°），在全图跨度 11°+ 的地图上两者几乎重叠，无法清晰展示两个独立城市节点。

## 目标

当相邻城市距离过近时，自动生成局部放大图，让用户看清该区域的细节。

---

## 设计决策

| 决策项 | 结论 |
|--------|------|
| 触发条件 | 自动，任意 segment 两端城市距离 < 全图对角线 × 2% |
| 放大内容 | 底图 + 省界 + 城市节点 + 路线段 + 景点（与主图风格一致） |
| 放大范围 | 自动：城市 + 景点 bbox × 150% 外扩 |
| 连通聚合 | A-B + B-C 同时过近 → 合并为一个放大区域 |
| 放大图位置 | 被放大区域附近的最近空白空间（螺旋搜索 + 碰撞检测） |
| 配置侵入 | 零，全自动 |

---

## 架构

```
加载配置
  → 计算地图范围
  → detect_close_pairs()          ← 新增：检测过近城市对
  → group_connected_pairs()       ← 新增：连通分量聚合
  → compute_zoom_extent()         ← 新增：计算每个放大区域的范围
  → 主图渲染（现有流程不变）
  → place_zoom_insets()           ← 新增：在空白空间放置放大图
  → render_zoom_insets()          ← 新增：渲染放大图 + 连接元素
  → 保存输出
```

## 文件改动

### 1. `config.py` — 新增常量

```python
INSET_THRESHOLD = 0.02          # 触发阈值（对角线的比例）
INSET_PADDING = 1.5             # 放大区域外扩倍数
```

### 2. `generate.py` — 新增函数

**`_detect_close_pairs(cities, segments, threshold)`**
- 计算所有城市坐标的 bbox 对角线长度
- 遍历 segments，检查 `distance(from_city, to_city) < threshold`
- 返回过近的城市索引对列表 `[(i, j), ...]`

**`_group_connected_pairs(pairs)`**
- 将过近对按连通关系聚合成组（union-find 或简单 BFS）
- 返回城市索引簇列表 `[{i, j}, {k, l}, ...]`

**`_compute_zoom_extent(cluster_cities, cities_dict, padding)`**
- 对每个簇，收集所有城市 + 景点的坐标
- 计算 bbox，按 padding 倍率外扩
- 确保宽高比合理（clamp 到 [0.5, 2.0]）
- 返回 extent `[lon_min, lon_max, lat_min, lat_max]`

**`_place_zoom_inset(layout, extent_zoom, cluster_center)`**
- 从 cluster_center 出发螺旋搜索候选位置
- 候选位置为放大图矩形的中心点
- 用 `LayoutEngine.rect_overlap()` 检测与已有元素的碰撞
- 找到第一个无冲突位置，返回 `(cx, cy, width, height)`（图坐标）

**`_render_zoom_inset(fig, ax_main, extent_zoom, cluster, position, ...)`**
- 在 position 处创建 `inset_axes`
- 设置 cartopy PlateCarree 投影
- 调用底图 + 省界 + 城市 + 路线 + 景点渲染函数
- 字体大小比主图缩小（×0.7）
- 在主图上绘制虚线矩形标记放大范围
- 绘制从虚线框到放大图的连接线

### 3. `renderer.py` — 新增函数

**`render_zoom_inset(ax, extent, cities, segments, cluster_indices, font_scale)`**
- 在新的 inset axes 上渲染局部底图
- 只绘制 cluster 内的城市和路线段
- 字体/标记缩小

**`render_zoom_indicator(ax_main, extent_zoom, inset_pos, linewidth, color)`**
- 主图上画虚线矩形（标记放大范围）
- 画 2 条连接线：虚线框一角 → 放大图对应角

### 4. 管道集成（`generate.py` → `generate()` 函数）

在 `_render_itinerary_and_title()` 之后插入：

```python
# L8: 局部放大图
close_pairs = _detect_close_pairs(cities, segments, threshold)
clusters = _group_connected_pairs(close_pairs)
for cluster in clusters:
    extent_zoom = _compute_zoom_extent(cluster, ...)
    position = _place_zoom_inset(layout, extent_zoom, cluster_center)
    _render_zoom_inset(fig, ax, extent_zoom, cluster, position, ...)
```

## 螺旋搜索算法

```
输入: cluster_center (lon, lat), inset_size (w, h), layout engine
输出: (cx, cy) 或 None

伪代码:
radius = 0.3  # 初始搜索半径(度)
for step in 1..20:
    for angle in [0, 90, 180, 270] + 45° 偏移:
        cx = center_lon + radius * cos(angle)
        cy = center_lat + radius * sin(angle)
        hw, hh = inset_size / 2
        if not layout.overlaps_anything(cx, cy, hw, hh):
            return (cx, cy)
    radius += 0.2  # 逐步扩大搜索半径
return None  # 回退到地图角落
```

## 非目标

- 不修改 JSON 配置格式
- 不修改 LayoutEngine 的核心算法
- 不处理非相邻城市的过近问题（只有 segment 中的直接连线对才检测）
- 不连通的城市簇各自生成独立的放大图（A-B-C连成一片→一个放大框；远距离的D-E→另一个）
- 放大图内不放行程表

---

## 回退策略

- 放不下时：按螺旋搜索结果，实在找不到位置就放到右上角
- 字体过小时：最小 6pt，低于此不渲染景点标签
- 所有异常吞掉，不影响主图生成
