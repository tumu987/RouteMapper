# 路线图生成引擎 — 设计文档

**日期**: 2026-06-01  
**目标**: 构建一个可复用的自驾路线图生成脚本，基于 Python + Cartopy + Matplotlib，输出高分辨率 PNG。

---

## 一、架构总览

三层分离架构：

```
数据层 (JSON 配置文件)
  ↓
布局引擎 (layout.py) — 碰撞检测 + 逐层放置 + 密度感知
  ↓
渲染引擎 (renderer.py) — 底图/路线/城市/标注/景点 各模块渲染
  ↓
主入口 (generate.py) — 解析配置 → 调用布局 → 调用渲染 → 输出 PNG
```

### 文件结构

```
行程路线图生成/
├── generate.py          # 主入口，解析参数和配置
├── layout.py            # 布局引擎（碰撞检测、逐层放置）
├── renderer.py          # 渲染引擎（底图、路线、城市、标注、景点）
├── config.py            # 配置加载与默认值
├── requirements.txt     # 依赖
├── routes/              # 路线配置文件
│   ├── nanjiang.json
│   ├── beijiang.json
│   ├── xian.json
│   └── laojunshan.json
└── output/              # 输出目录（可配置）
```

## 三、布局引擎设计

### 3.1 放置顺序（6 层）

| 层 | 内容 | 策略 |
|----|------|------|
| L1 | 底图（陆地、水域、省界、经纬网） | 地理坐标固定 |
| L2 | 路线线条 | 地理坐标固定，注册线宽到碰撞系统 |
| L3 | 城市节点（圆圈 + 圈内文字） | 地理坐标固定，同纬度自动微调 |
| L4 | 天次标注（D1-DN） | 密度感知侧选 + 递进搜索 |
| L5 | 距离/时间标注 | 主动避让天次，优先去对侧 |
| L6 | 景点引线 + 行程表 | 8 方向寻空 + 网格搜索 |

### 3.2 碰撞检测系统

三种检测方式：

1. **矩形 vs 矩形**：文字标签之间的碰撞，分别计算半宽半高
2. **矩形 vs 路线段**：点到线段最短距离，考虑路线线宽
3. **对侧宽松**：天次和距离/时间在路线对侧时，margin 降至 0.01°

### 3.3 文字尺寸估计

```
半宽 ≈ 字符数 × 字号(pt) × 0.0031 + 0.005
半高 ≈ 字号(pt) × 0.0036 + 0.005
```

### 3.4 密度感知侧选

对于天次标注和距离/时间，选择放置侧时：
1. 计算路线法线方向两侧的偏移位置
2. 统计每侧一定半径内已放置元素数量
3. 优先选元素少的稀疏侧

### 3.5 递进搜索

同一个侧内，从最近距离开始尝试：
```
base_offset → 1.5× → 2× → 2.5× → 3× → fallback (8方向)
```

### 3.6 行程表自动定位

网格搜索地图范围内最大空白矩形：
1. 将地图分成 0.15° 网格
2. 评分 = 附近已放元素数 + 距顶边距离（偏好上方）
3. 选最低分位置
4. 如单列放不下，自动拆双列

## 四、渲染引擎设计

### 4.1 底图渲染

- 投影：PlateCarree（等距圆柱）
- 陆地：`#F8F4EC`，水域：`#E8EDF4`
- 湖泊：`#DAE5F0` 填充，`#AABBCC` 边框
- 海岸线：`#999999`，线宽 1.2
- 经纬网：虚线，只显示左下刻度
- 省界：途经省份，各自颜色，线宽 2.0，透明度 0.25
- 省份名：半透明水印，透明度 0.30

### 4.2 路线渲染

多层发光羽化效果：
```
宽22 × alpha 0.06 → 宽16 × alpha 0.12 → 宽10 × alpha 0.25 → 核心线宽6
```
每天独立颜色，相邻天颜色有足够区分。支持任意天数。

### 4.3 城市节点

- 正圆（椭圆补偿纬度拉伸）
- 白色填充 + 省份色边框（线宽 3）
- 城市名在圈内居中，大号加粗华文黑体，省份色
- 同纬度城市手动微调纵坐标

### 4.4 标注渲染

| 类型 | 字体 | 大小 | 颜色 | 描边 |
|------|------|------|------|------|
| 主标题 | 华文黑体加粗 | 30pt | #2C3E50 | 白色6px |
| 副标题 | 华文黑体加粗 | 18pt | #7F8C8D | 白色4px |
| 天次 D1-DN | 华文黑体加粗 | 18pt | 紫色 #8B44AC | 白色4px |
| 距离 | 华文黑体加粗 | 16pt | 蓝色 #2980B9 | 白色4px |
| 时间 | 华文黑体加粗 | 14pt | 橙色 #E67E22 | 白色4px |
| 主要景点 | 华文黑体加粗 | 14pt | 棕色 #8B4513 | 白色3.5px |
| 次要景点 | 华文黑体加粗 | 13pt | 绿色 #27AE60 | 白色3.5px |
| 行程表 | 华文黑体加粗 | 14pt | #2C3E50 | 白色1.5px |

全部文字加粗 + 白色描边。

### 4.5 景点引线

- 主要景点：实线引线 + 实心圆点
- 次要景点：虚线引线 + 空心圆点
- 城圈内景点：合并引线拉出，纵向排列
- 城圈外景点：真实坐标打点 + 紧邻标签
- 左右交替布局，自然分隔

## 五、代码设计原则

### 5.1 与现有脚本的关系

- 整合 `nanjiang_v3.py` 的矩形碰撞引擎和密度感知逻辑
- 整合 `generate_final.py` 的行程表自动定位和景点分组策略
- 对齐 `generate_route_v5.py` 的省界颜色区分
- 整体结构从头重写，数据与渲染彻底分离

### 5.2 编程规范

- Python 3.9+
- 依赖：matplotlib, cartopy, numpy, shapely
- 4 空格缩进，类型注解
- 函数单一职责，每个函数不超过 100 行
- 布局引擎不依赖渲染，可单元测试

### 5.3 可扩展性

- 新增路线：只需添加 JSON 配置文件
- 自定义样式：配置文件可覆盖默认字体/颜色/尺寸
- 自定义布局策略：布局引擎提供回调钩子

## 六、输出规格

| 属性 | 规格 |
|------|------|
| 格式 | PNG |
| 默认尺寸 | 22×26 英寸 @ 120 DPI (2640×3120 px) |
| 背景 | 纯白 |
| 缓存路径 | `~/.hermes/cache/documents/{路线名}.png` |
| 桌面副本 | `~/Desktop/{路线名}.png` |
| 文件名 | `{起点}_{终点}_{标题}.png` |

### 自动图幅扩展

当地图范围不够容纳行程表时，自动向北扩展北边界。扩展量 = 行程表所需高度 + 安全边距。

## 八、验收标准

1. `nanjiang_config.json` 能生成符合需求书全部规格的路线图
2. 图中所有元素不重叠（天次、距离、时间、景点、城市圈之间）
3. 已有4条路线配置全部可运行出图
4. 新增一条路线只需编写 JSON 文件，不改代码
5. 输出分辨率 ≥ 2640×3120 px，PNG 格式
