## 一、全局决策

| # | 问题 | 决策 |
|---|------|------|
| 1 | 模板化 | 数据与渲染分离，JSON配置驱动，可复用 |
| 2 | 起手方式 | 从零重写，整合 nanjiang_v3 + generate_final 优点 |
| 3 | 布局策略 | 分优先级逐层放置 + 密度感知 |
| 4 | 行政区划 | 省界（Natural Earth），暂不细化到市/区县 |
| 5 | 路线视觉 | 清晰即可，不过粗，4层发光羽化，自动调色板分配 |
| 6 | 城市圈大小 | 统一半径，长名超出圈外，白色描边保可读 |
| 7 | 城市圈颜色 | 按省份统一色，自动检测 |
| 8 | 天次位置 | 密度感知选侧，优先路线外侧 |
| 9 | 距离/时间 | 天次对面优先，空间不够同侧也可 |
| 10 | 景点引线 | 默认无引线，仅空间不够才加 |
| 11 | 景点排列 | 纵向堆叠，最紧凑，不固定侧 |
| 12 | 景点上限 | 每城最多 2 主要 + 2 次要 |
| 13 | 同日多段天次 | 标在总和的中间位置 |
| 14 | 同日多段距离 | 每段分别标 |
| 15 | 长城市名 | 超出圈外，白色描边 |
| 16 | 连续休息日 | 合并标注 |
| 17 | 路线穿城市 | 自然穿过，城市圈zorder更高遮住 |
| 18 | 主次景点区分 | 字号+颜色（棕主/绿次） |
| 19 | 省份颜色 | 保持预设表 |
| 20 | 极短路段 | 正常标注，碰撞检测外推 |
| 21 | 行程表内容 | 自动换行 |
| 22 | 输出格式 | 仅PNG |

## 三、放置顺序

```
L1 底图+省界（固定）
L2 路线线条（固定，注册碰撞系统）
L3 城市节点（固定坐标）
L4 景点标注（优先级高于天次）
L5 天次标注
L6 距离/时间标注
L7 行程表（最后，自动找空位）
```

---

## 四、景点系统

### 城圈内景点
- 折线引线连接城圈：城圈边缘→斜线→拐点→水平线→文字
- 每城共享一条引线
- 纵向堆叠，行间距紧凑

### 城外景点
- 坐标点打圆点标记
- 默认无引线，文字紧邻坐标
- 距离较远时用折线引线

---

## 五、核心约束

- 所有元素不得重叠（文字、路线、城市圈之间）
- 天次、距离、时间紧贴路线
- 当"靠近线路"和"不压线"冲突时，不压线优先
